## Steps

### 1. Read .env.example
Read the `.env.example` file at the project root. Parse all variable names and placeholder values.

### 2. Generate Safe Development Defaults

For each variable, apply these rules:

**Database**:
- `DATABASE_URL` → `postgresql://postgres:postgres@localhost:5432/amazone_dev`

**Auth (NextAuth)**:
- `NEXTAUTH_SECRET` → Generate a random 32-character hex string using: `node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"`
- `NEXTAUTH_URL` → `http://localhost:3000`
- `AUTH_GOOGLE_ID` → `google-oauth-dev-id` (placeholder)
- `AUTH_GOOGLE_SECRET` → `google-oauth-dev-secret` (placeholder)

**Stripe** (test mode placeholders):
- `STRIPE_SECRET_KEY` → `sk_test_placeholder_replace_with_your_key`
- `STRIPE_PUBLISHABLE_KEY` → `pk_test_placeholder_replace_with_your_key`
- `STRIPE_WEBHOOK_SECRET` → `whsec_placeholder_replace_with_your_key`

**Redis**:
- `REDIS_URL` → `redis://localhost:6379`

**Next.js**:
- `NEXT_PUBLIC_APP_URL` → `http://localhost:3000`

### 3. Apply User Overrides
If $ARGUMENTS contains key=value pairs, apply them to override the defaults.

### 4. Write .env.local
Write the file to `.env.local` at the project root with a header comment:
```

# DO NOT commit this file (it's in .gitignore)
```

- Confirm `.env.local` is in `.gitignore`
